# serial/

## 責務

- シリアルポートの一覧取得(`list`)・接続(`open`)・切断(`close`)・書き込み(`write`)・受信(`onData`)のみを提供する。
- `mockTransport.ts` は実GRBLデバイスの代わりに使うテスト用スタブ(`ok`/`error:N`/ステータス報告を文字列で返す)。

## やってはいけないこと(不変条件)

- GRBLのプロトコル構文(`ok`/`error:N`/`<...>`等)を一切解釈しないこと。生データの送受信のみを行う。
- このモジュール以外からシリアルポートへ直接書き込まないこと。書き込みは必ず `grbl/scheduler.ts` 経由。
- 改行によるフレーミング(行の分割・バッファリング)もここでは行わない。それは `grbl/parser.ts` の責務。
